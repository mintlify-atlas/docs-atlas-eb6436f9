---
title: "Quickstart"
description: "Get a GameTemplate simulation running from scratch."
---

<Steps>

  <Step title="Create your repository">
    Use the **Use this template** button on GitHub, or create a new repo via the CLI:

    ```bash
    gh repo create --template Ignaciotema7b/GameTemplate MyGame
    ```

    After your first push to `main`, the `setup-template.yml` workflow fires automatically. It replaces every occurrence of `GameTemplate` — in file contents, file names, and directory names — with your repository name, commits the result as `Auto-rename project to MyGame`, and then removes itself. Your project is now fully renamed without any manual find-and-replace.

    <Note>
      The workflow skips the rename if the repository name is still `GameTemplate`, so it is safe to work in a fork of the original template itself.
    </Note>
  </Step>

  <Step title="Open the solution">
    Open `GameTemplate.sln` (renamed to `MyGame.sln` after the workflow runs) in **Visual Studio 2022** or later. The solution targets **C++17** and uses `std::filesystem`, so make sure your toolchain supports it.

    The solution ships with several pre-configured build configurations:

    | Configuration | Purpose |
    |---|---|
    | `Debug` / `Release` | Standard builds |
    | `z_Logs_Debug` / `z_Logs_Release` | Enables `LOG_PLAYS` for random logging |
    | `z_Proto_Debug` / `z_Proto_Release` | Enables `LOG_PROTO` for prototype logging |
    | `z_Synchro_*` | Enables `WINS_SYNCHRO` for synchronised win tracking |

    <Tip>
      For large simulation runs, always build in a `Release` configuration to get full compiler optimisations.
    </Tip>
  </Step>

  <Step title="Prepare configuration files">
    The simulation reads all game parameters from CSV files at startup. Place these files in `game/config/` (relative to the working directory where you run the executable).

    <Warning>
      `info.csv` must be present before the simulation can start. If the file is not found, `Game_info::load_info()` will fail to initialise and the program will not run.
    </Warning>

    **`info.csv`** — core simulation parameters (field names are case-sensitive):

    | CSV field name | Description |
    |---|---|
    | `Game name` | Used in the output JSON file name |
    | `Version` | Config version string, also appears in the output file name |
    | `Type` | Game mechanic: `Paylines`, `Ways`, `Scatter pays`, `Clusters`, `Clusters (8 way)` |
    | `Number of reels` | Number of vertical reels (excludes horizontal trackers) |
    | `Number of rows` | Maximum visible rows (screen height) |
    | `Simulation plays` | Total iterations to simulate |
    | `Stake` | Stake value in credits |
    | `Cap` | Maximum win multiplier for regulatory capping |
    | `Seed` | RNG seed; set to `0` for a random seed each run |
    | `Mode` | Descriptive label printed during simulation (e.g. `regular`) |

    **`reel_set__0.csv`**, `reel_set__1.csv`, … — reel strips. Each row in the file is one reel strip, with comma-separated symbol IDs. Multiple reel sets are supported; `w_reel_sets_main` controls the weighted selection between them.

    **`symbols.csv`** — symbol definitions: ID, short name (≤ 3 characters), full name.

    **`paytable.csv`** — pay values. Rows correspond to symbols, columns to the number of matching symbols in a winning combination.

    **`paylines.csv`** *(Paylines games only)* — one row per payline. Each column holds the row position for that payline on the corresponding reel.

    <Tip>
      The executable also falls back to looking for config in a `config/` directory (without the `game/` prefix), so you can run it directly from the `game/` folder during development.
    </Tip>
  </Step>

  <Step title="Implement game logic">
    Open `src/GameTemplate.cpp` (renamed to `src/MyGame.cpp`). The `play()` method already handles the simulation loop, screen reset, reel-set selection, and reel spinning. Add your win-evaluation logic where the comment reads `//...write game logic here`:

    ```cpp
    void GameTemplate::play(unsigned long long plays)
    {
        unsigned using_reel_set;
        unsigned long long base_win;
        unsigned long long new_win;
        unsigned total_win;

        for (unsigned long long iter = 0; iter < plays; iter++)
        {
            // Periodic progress print
            if (iter > 0 && !(iter % plays_to_print))
            {
                system("CLS");
                cout << endl << "Play mode: " << info.mode << endl;
                stats.print_stats(stake);
            }

            // Reset win accumulators and screen
            base_win = 0;
            new_win  = 0;
            total_win = 0;
            clear_screen();

            // Select a reel set (weighted) and spin
            using_reel_set = config.w_reel_sets_main.sample(rng);
            config.reel_sets[using_reel_set].spin(rng);
            drop_new_symbols(config.reel_sets[using_reel_set]);

            // --- Add your win evaluation here ---
            // Example for a Ways game:
            // base_win = get_ways_win(config);

            // Cap wins for regulatory compliance
            // cap_wins(base_win);

            total_win = base_win;

            // Accumulate statistics
            stats.count_stats_play(total_win, stake);
        }

        // Write JSON results file
        generate_json(info);

        system("CLS");
        cout << endl << "Play mode: " << info.mode << endl;
        stats.print_stats(stake);
    }
    ```

    `cap_wins()` is variadic — you can pass multiple win components and it enforces `cap × bet_for_cap` across their sum in argument order:

    ```cpp
    // Cap base_win first, then bonus_win, trimming whatever exceeds the cap
    cap_wins(base_win, bonus_win);
    ```
  </Step>

  <Step title="Build and run">
    Build the solution in Visual Studio, then run the executable from the `game/` directory (or any directory that has a `game/config/` subfolder containing `info.csv`).

    The `main()` function in `GameTemplate_main.cpp` will:

    1. Locate `game/config/info.csv` (falls back to `config/info.csv`).
    2. Call `Game_info::load_info()` to parse all CSV files.
    3. Seed the `mt19937_64` RNG — fixed seed if `info.seed != 0`, random otherwise.
    4. Call `game->play(info.number_of_plays)` to run the full simulation.
    5. Print execution time and pause.
  </Step>

  <Step title="Review results">
    After the simulation completes, a JSON file is written to `game/results/` (created automatically if it does not exist). The file is named:

    ```
    _rtp_[rtp]__[unix_timestamp]__[game_name]_v[version].json
    ```

    For example:

    ```
    _rtp_0.960123__1742900400__MyGame_v1.0.json
    ```

    The file contains three top-level keys:

    | Key | Contents |
    |---|---|
    | `config` | All loaded configuration parameters (`config.json_config`) |
    | `info` | Game info fields including the actual RNG seed used (`rng_seed`) |
    | `stats` | Full simulation statistics from `stats.stats_to_json(stake)` |

    <Note>
      The UNIX timestamp in the file name is the simulation finish time. It is not stored inside the JSON body — only in the file name.
    </Note>
  </Step>

</Steps>
