## CommonLib вЂ” the framework library

`CommonLib/` is the reusable engine. Its `include/` and `src/` directories mirror each other with one header and one `.cpp` per class.

### Core classes

| Class | Role |
|---|---|
| `Game` | Base class: MT19937-64 RNG, `stake`, `cap`, weighted sampling (`sample`, `sample_index`, `sample_rc`), variadic `cap_wins()` |
| `Slot` | Extends `Game`: screen array, reel-set management, `clear_screen()`, `drop_new_symbols()`, `get_ways_win()`, `get_lines_win()`, `generate_all_symbols_combinations()` |
| `Slingo` | Extends `Game`: Slingo-specific board and number-drawing logic |
| `Slot_config` | Loads and owns all configuration data: reel sets, symbol list, paytable, paylines, JSON config representation |
| `Reel_set` | Holds one set of reel strips and exposes `spin(rng)` to pick random stops |
| `Symbol` | Stores symbol ID, short name (в‰¤ 3 chars), and full name |
| `Paylines` | Parses `paylines.csv` and evaluates winning lines on the screen |
| `Prize` | Represents a single prize: symbol, count, and pay value |
| `Weighted_values` | Efficient weighted random sampling; use `sample(rng)` for repeated draws |
| `Game_statistics` | Accumulates simulation metrics: total payout, play count, hit frequency, RTP; exposes `count_stats_play()`, `print_stats()`, `stats_to_json()` |
| `Game_info` | Parses `info.csv` and holds all runtime parameters (`n_reels`, `number_of_plays`, `bet`, `cap`, `seed`, `mode`, etc.) |
| `rnd_log` | Records per-play RNG state for debugging; only compiled in when `LOG_PLAYS` is defined |
| `Win_and_RTP_distribution` | Tracks the full distribution of win amounts for histogram output in results |

## src/ вЂ” your game implementation

| File | Role |
|---|---|
| `GameTemplate.h` / `GameTemplate.cpp` | The game class itself. Inherits from `Slot`. Contains `init()`, `play()`, and `generate_json()`. This is where you write spin and win-evaluation logic. |
| `GameTemplate_config.h` / `GameTemplate_config.cpp` | Extends `Slot_config`. Owns `w_reel_sets_main` (`Weighted_values`) and any additional config state your game needs. |
| `GameTemplate_stats.h` / `GameTemplate_stats.cpp` | Extends `Game_statistics`. Override `stats_to_json()` and `print_stats()` to add game-specific stat fields. |
| `GameTemplate_main.cpp` | Entry point (`main()`). Locates config files, calls `load_info()`, seeds the RNG, runs `play()`, and reports execution time. |

```
Game
в”њв”Ђв”Ђ Slot в”Ђв”Ђв”Ђ GameTemplate
в”‚               в”њв”Ђв”Ђ GameTemplate_config  (extends Slot_config)
в”‚               в””в”Ђв”Ђ GameTemplate_stats   (extends Game_statistics)
в””в”Ђв”Ђ Slingo
```

`GameTemplate` holds a `GameTemplate_config config` and `GameTemplate_stats stats` as public members. `init()` delegates to `config.init(info)` and `Slot::init(...)`, then seeds stats and optional logging. `play()` runs the simulation loop and finishes by calling `generate_json(info)` to write results.

## The game/ directory

The `game/` directory is the **runtime working directory** for the simulation executable. It contains only a `.gitkeep` in the template; you populate it before running:

```
game/
в”њв”Ђв”Ђ config/           в†ђ place all CSV files here
в”‚   в”њв”Ђв”Ђ info.csv
в”‚   в”њв”Ђв”Ђ reel_set__0.csv
в”‚   в”њв”Ђв”Ђ symbols.csv
в”‚   в”њв”Ђв”Ђ paytable.csv
в”‚   в””в”Ђв”Ђ paylines.csv  в†ђ Paylines games only
в””в”Ђв”Ђ results/          в†ђ created automatically by generate_json()
```

The executable first looks for `game\config\info.csv`. If not found it falls back to `config\info.csv`, so you can also run it from inside `game/` directly.

## Compile-time flags

Three preprocessor flags change build behaviour. They map to the named solution configurations in `GameTemplate.sln`:

| Flag | Solution config prefix | Effect |
|---|---|---|
| `LOG_PLAYS` | `z_Logs_*` | Compiles in the `rnd_log randomLogs` member and calls `randomLogs.init(info)` inside `init()`. Records per-play RNG state for debugging and reproducibility. |
| `LOG_PROTO` | `z_Proto_*` | Enables prototype logging output during simulation. |
| `WINS_SYNCHRO` | `z_Synchro_*` | Enables synchronised win tracking, useful for multi-threaded or deterministic comparison runs. |

All three flags are defined only in their respective configurations вЂ” standard `Debug` and `Release` builds compile none of them.
