GameTemplate is a C++ framework developed by **7 Bridges Maths** for rapid development of casino game math models. It provides a complete set of reusable classes for reel mechanics, paytables, win evaluation, and statistical analysis — so you can focus on game-specific logic rather than boilerplate infrastructure.

## How it works

<Steps>
  <Step title="Scaffold your game">
    Use the GitHub template to create a new repository. The setup workflow automatically renames all files and classes to match your game name.
  </Step>
  <Step title="Configure game parameters">
    Place CSV configuration files (reel strips, paytable, paylines, symbol list) in the `game/config/` directory. Load them through `Game_info::load_info()`.
  </Step>
  <Step title="Implement game logic">
    Override the `play()` method in your derived class. Call `Slot` methods to spin reels, evaluate wins, and apply features.
  </Step>
  <Step title="Run Monte Carlo simulation">
    Compile and run. The simulation executes the configured number of plays, tracks full statistics, and writes a JSON results file with RTP and win distributions.
  </Step>
</Steps>

## Key features

- **MT19937-64 RNG** — 64-bit Mersenne Twister with seed control for fully reproducible simulations
- **Multiple win modes** — Paylines, Ways, Scatter pays, Clusters (4-way and 8-way), and bidirectional Ways
- **Win capping** — Variadic `cap_wins()` for regulatory compliance across multiple win sources
- **Weighted sampling** — Efficient `Weighted_values` class with cumulative weights and sampling without replacement
- **Cascade mechanics** — Built-in symbol explosion and gravity drop for tumble/cascade games
- **Symbol multipliers** — Per-cell multiplier grids for bonus mechanics
- **JSON output** — Structured results files with configuration snapshot, simulation info, and full statistics
- **Template scaffold** — GitHub Actions workflow renames the entire project automatically when you create a new repo from the template

<Tip>
  The `game/` directory is intentionally empty in the template. Add your `config/` CSV files there, and optionally a `results/` directory for simulation output.
</Tip>
